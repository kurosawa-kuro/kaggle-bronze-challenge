# 01 要件

## 目的

Kaggle コンペティションでブロンズメダル（上位 40% 前後）を獲得する。  
表形式データに対して再現性の高いパイプラインを手元環境で構築し、提出スコアを安定させる。

## スコープ

### 対象

- 表形式データを扱う Kaggle コンペへの参加・提出
- 前処理 → 学習 → 推論 → 提出ファイル生成のパイプライン構築
- 特徴量エンジニアリングとモデル選択の反復実験

### 非対象

- シルバー・ゴールドメダル水準の最終 push（アンサンブル過多、リーク活用等）
- 画像・音声・テキスト主体のコンペ（Deep Learning への過度な展開は行わない）
- Google Colab 環境での動作保証

## 制約・方針

| 区分 | 内容 |
|------|------|
| **目標水準** | ブロンズ取得。シルバーは狙わない |
| **モデル選択** | 勾配ブースティング系（CatBoost / LightGBM）を主軸。Deep Learning は補助的に留める |
| **フレームワーク** | scikit-learn パイプライン互換で統一 |
| **レシピ参照元** | Databricks のデザインパターン・ノートブックレシピを優先参照 |
| **実行環境** | WSL Ubuntu（ローカル実行）|
| **ストレージ** | 実験ログ・中間データ → SQLite。集計・分析 → DuckDB または BigQuery |

## 技術スタック

```
アルゴリズム : CatBoost, LightGBM 、XGBoost
フレームワーク: scikit-learn
DB           : SQLite（実験トラッキング・軽量ストレージ）
DWH          : DuckDB（ローカル）or BigQuery（大規模集計時）
環境         : WSL Ubuntu / Python
仮想環境は使うべきか？ない方が高速に回せるか？
```

## ユーザー

- コンペ参加者（本人のみ）

## ユースケース

| ID | ユースケース | 成功条件 |
|---|---|---|
| UC-001 | 新規コンペのデータを取得し EDA を実施する | データ概要・欠損・分布を把握したノートブックが生成される |
| UC-002 | ベースラインモデルを学習・提出する | Public LB スコアが記録され、ブロンズ圏内かどうかを確認できる |
| UC-003 | 特徴量エンジニアリングを反復して改善する | 実験ごとのスコアが SQLite に記録され比較できる |
| UC-004 | 最終提出ファイルを生成する | `submission.csv` が Kaggle 規格に適合している |
